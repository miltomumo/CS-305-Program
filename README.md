# CS-305-Program
My client for this project was Artemis Financial, a financial company that focuses on providing custom customer budgeting solutions. The goal of the project was to help Artemis Financial modernize its operations and improve security. For this specific software, the company needed me to analyze existing vulnerabilities and apply secure communication mechanisms.
  I began by creating a digital certificate to ensure secure communications between servers, and I implemented a SHA‑256 hashing algorithm to encrypt sensitive information such as passwords and personal data. In addition, I conducted both manual code reviews and used automated tools to confirm that no new vulnerabilities were introduced during refactoring.
  The most challenging part of the vulnerability assessment was reading and interpreting the automated report. I had to learn how to distinguish between false positives and true vulnerabilities, and how to suppress false positives so the report only flagged issues that required fixing. This process helped me develop a stronger understanding of how to evaluate and prioritize security risks.
  I increased the security of Artemis Financial’s application by implementing the SHA‑256 encryption algorithm to protect sensitive data, generating a certificate, and enforcing HTTPS to ensure trusted and secure communications. Finally, I ran an OWASP Dependency‑Check to verify that no new vulnerabilities were introduced. In the future, I plan to continue utilizing dependency checks for automated vulnerability assessment, while also performing manual reviews to ensure comprehensive coverage.
  To make certain the application remained functional and secure, I compiled the code frequently to avoid introducing errors. I also tested the HTTPS endpoint in a browser and validated checksum outputs to confirm both functionality and integrity.
  Some resources and practices I will apply in future assignments include the OWASP Dependency‑Check for static analysis, since it provides detailed vulnerability reports along with remediation suggestions, and checksum verification to test file integrity.
  For future employers, I would showcase my ability to review and refactor existing code using both automated and manual techniques, my implementation of secure communication mechanisms such as hashing, HTTPS, and certificates, and my ability to produce a vulnerability assessment report that demonstrates my skills in identifying, interpreting, and mitigating security risks.
